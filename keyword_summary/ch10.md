## 10주차 핵심 키워드
- 클라우드 컴퓨팅이란?

  클라우드 컴퓨팅은 서버, 스토리지, 데이터베이스와 같은 IT 자원을 인터넷을 통해 필요한 만큼 빌려 사용하는 방식이다.

  과거에는 기업이 직접 서버를 구매하고 관리해야 했지만, 클라우드 환경에서는 필요한 시점에 필요한 만큼만 사용하고 비용을 지불할 수 있다.

  **장점**

    - 초기 서버 구축 비용 절감
    - 필요에 따라 자원 확장 가능
    - 장애 대응 및 유지보수가 편리함
    - 전 세계 어디서든 서비스 가능

  **단점**

    - 인터넷 연결에 의존적
    - 장기적으로는 비용이 증가할 수 있음
    - 클라우드 제공 업체에 대한 의존성 존재
- AWS? GCP?

  **AWS (Amazon Web Services)**

  아마존에서 제공하는 클라우드 플랫폼으로 가장 많은 사용자를 보유하고 있다.

  **특징**

    - 다양한 서비스 제공
    - 방대한 문서와 레퍼런스
    - 높은 시장 점유율
    - 기업에서 가장 많이 사용

  **GCP (Google Cloud Platform)**

  구글에서 제공하는 클라우드 플랫폼이다.

  **특징**

    - 빅데이터 및 AI 서비스 강점
    - 구글 서비스와 연동 용이
    - 직관적인 UI 제공

- 환경변수 처리 방법과 왜 환경변수로 민감 정보를 가려야 하는가?

  환경변수는 프로그램 실행 시 필요한 설정값을 코드 외부에서 관리하는 방법이다.

  예를 들어:

    - DB 비밀번호
    - API Key
    - JWT Secret Key
    - AWS Access Key

  **올바른 예시**

    ```jsx
    password: ${DB_PASSWORD}
    
    jwt:
    	secret: ${JWT_SECRET}
    ```

  **환경변수를 사용하는 이유**

    1. 보안 강화
    2. GitHub 업로드 시 민감 정보 노출 방지
    3. 개발/운영 환경 분리 가능
    4. 설정 변경 시 코드  수정 불필요
- yml 환경 분리 방법

  ### application.yml

    ```
    spring:
      profiles:
        active: dev
    ```

  ### application-dev.yml

    ```
    server:
      port: 8080
    ```

  ### application-prod.yml

    ```
    server:
      port: 80
    ```

  ### 환경 분리 장점

    - 개발 환경(dev)
    - 테스트 환경(test)
    - 운영 환경(prod)
- Docker와 .jar vs Docker 이미지

  ### .jar 파일

  Spring Boot 프로젝트를 빌드하면 생성되는 실행 파일이다.

    ```
    java-jar app.jar
    ```

  만으로 실행 가능하다.

  ### 장점

    - 생성이 간단함
    - 파일 크기가 작음

  ### 단점

    - Java 설치 필요
    - 서버 환경마다 설정이 달라질 수 있음

  ### Docker 이미지

  애플리케이션과 실행 환경을 함께 패키징한 이미지이다.

  ### 장점

    - 어느 서버에서든 동일한 환경 보장
    - 배포 과정 단순화
    - 컨테이너 기반 관리 가능
    - MSA 환경에 적합

  ### 단점

    - Docker 학습 필요
    - 이미지 용량 증가
    - 컨테이너 관리 필요

  ### .jar vs Docker 이미지

  | 항목 | .jar | Docker 이미지 |
      | --- | --- | --- |
  | 실행 환경 | 서버에 Java 설치 필요 | Docker만 있으면 실행 가능 |
  | 배포 편의성 | 보통 | 높음 |
  | 환경 일관성 | 낮음 | 높음 |
  | 운영 적합성 | 소규모 프로젝트 | 실무 서비스 |
